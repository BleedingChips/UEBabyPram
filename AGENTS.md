# AGENTS.md

## Build system

- **xmake** is the build system. Use `xmake build`, `xmake run <target>`, etc.
- `xmake.lua` is at the repo root. The `vsxmake2022/` directory contains auto-generated VS solution/project files. **Do not edit** `.sln` or `.vcxproj` files in `vsxmake2022/` — they are regenerated by `xmake project -k vsxmake`.
- To regenerate VS files: `.\xmake_generate_vs_project.ps1`

## Repo structure

```
UEBabyPram/              # repo root (where xmake.lua lives)
  xmake.lua
  UEBabyPram/            # static lib: UEBabyPram + UEBabyPramLogParser
  UEBabyPramLogFilter/   # binary: UE log CLI tool (depends on re2/abseil)
  Test/                  # test binaries (one target per .cpp file)
  vsxmake2022/           # auto-generated VS solution (gitignored)
  build/                 # build output (gitignored)
```

The repo depends on `Potato` (a sibling repo — must be cloned at `../Potato/` alongside this one).

## Targets

| Target              | Kind   | Dependencies                         |
|---------------------|--------|--------------------------------------|
| UEBabyPram          | static | Potato, ctre                         |
| UEBabyPramLogFilter | binary | Potato, UEBabyPram, re2 (and abseil) |
| LogFilterTest       | binary | UEBabyPram, Potato                   |

Test targets are auto-generated from each `.cpp` file in `Test/` with the filename as the target name.

## Configurations

`debug`, `release`, `releasedbg`, `profile` — all Windows x64 (MSVC).

## Language

- C++ latest (`cxxlatest` in xmake)
- Uses C++20 modules (`.ixx` files). Module files use `export module Foo;` / `export import Foo;`. Non-module files use `import Foo;`.
- Source uses `std::u8string_view` from Potato.
- The `Potato` library provides the `std` module and foundational utilities.

## xmake.lua guard pattern

The xmake.lua wraps most targets in `if os.scriptdir() == os.projectdir() then ... end` so this repo can be pulled in as a subproject (`includes("../UEBabyPram/")`) without re-defining targets that only make sense as top-level.

## How to test

Each `.cpp` in `Test/` becomes a target. Build and run directly:
```
xmake build LogFilterTest
xmake run LogFilterTest
```

## Dependencies

- `Potato`: sibling repo, referenced via `includes("../Potato/")`
- `ctre`: compile-time regex, xmake package
- `re2` + `abseil`: only needed by `UEBabyPramLogFilter` binary
